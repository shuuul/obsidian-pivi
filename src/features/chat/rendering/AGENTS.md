**Purpose**: Contains the DOM rendering subsystem for the Obsius chat feature. It takes structured chat messages, tool calls, subagent states, diff blocks, and thinking content — and converts them into accessible, interactive Obsidian DOM elements with collapsible sections, copy buttons, and progress indicators.

**Key Files**:
- `MessageRenderer.ts` — Main orchestrator class that renders entire conversation threads, dispatching to specialized renderers for each content block type (thinking, tool_use, subagent, text)
- `collapsible.ts` — Shared utility implementing click/keyboard/ARIA expand/collapse behavior, reused by ThinkingBlock, Subagent, and ToolCall renderers
- `SubagentRenderer.ts` — Renders sync and async subagent blocks with collapsible prompt, tool calls, and result sections; supports streaming state updates (`createSubagentBlock` → `finalizeSubagentBlock`)
- `DiffRenderer.ts` — Renders inline diff hunks with line-by-line insert/delete visualization, hunks separated by `...`, and new-file caps (20 lines)
- `ThinkingBlockRenderer.ts` — Renders collapsible "Thinking Xs..." blocks with a live timer during streaming and final duration on completion

**Patterns**:
- **Streaming vs stored dual paths**: Every renderer has separate `create*` (live) and `renderStored*` (replay) functions, enabling incremental DOM updates during streaming and batch rendering from history
- **Shared collapsible utility**: `setupCollapsible()` from `collapsible.ts` powers expand/collapse across thinking, subagent, and tool call blocks — with ARIA attributes, keyboard nav, and CSS class management
- **Provider lifecycle delegation**: `subagentLifecycleResolution.ts` bridges to provider-specific lifecycle adapters via `ProviderRegistry`, keeping rendering logic provider-neutral per the hexagonal architecture
